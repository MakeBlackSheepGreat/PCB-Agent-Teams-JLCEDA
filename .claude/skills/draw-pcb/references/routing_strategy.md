# 布线策略 — KRT 配方的逐网判断框架

`route` 工具把 A* 寻路做成机械执行;**线宽多少、哪些网走粗、差分对怎么处理、布线顺序选哪个**是电路判断,由 AI 按本文先给 net 分类再定配方。跟 draw-pcb 总分工一致:判断走回路,机械走脚本。

> 反模式:把 `route` 当裸 `route <pcb>` 一键跑,~46 个 KRT 参数全吃默认。默认 track width 很细(~0.1mm),对电源配电干线偏窄;差分对不设阻抗就当普通网布。

## 先分类,再定配方

布线前对板上的网做一次分类——这一步必须有可见输出(打印每类有哪些网):

| net 类 | 怎么认 | 配方 |
|---|---|---|
| **信号网** | 一对一 / 少 pad,小电流 | `--track-width` base 值;通常吃 KRT 默认即可 |
| **电源网** | 配电干线 / 多 pad 供电网 / 已知较大电流 | 进 `--power-nets`,`--power-nets-widths` 给加宽线 |
| **差分对** | 名字成对(`*_P`/`*_N`)、需阻抗匹配(如 ADC / 高速) | `--impedance` 设目标阻抗;P/N 尽量同层并行 |
| **GND / 已铺铜网** | 已由 `add_zones` 铺铜面 | 见 routing.md:KRT 仍会布 GND 走线,冗余无害;要省可 `--nets` 排除 |

**不需要配方就别配**:一块全是小信号、无电源干线、无差分对的简单板,裸 `route <pcb>` 吃默认就对——不要为了用参数而用。配方是给"有功率网 / 有差分对 / 布不通"的板。

## 各轴判断

### 线宽(--track-width / --power-nets-widths)
- 信号网:base 宽度够载流即可,默认偏细但短走线无妨。
- 电源网:线宽由**电流**定,不是拍脑袋。方向性参考 IPC-2221 外层经验(约 0.5mm/A @ 10℃ 温升),但实际取值还要看 fab 最小线宽能力和走线长度(IR 压降)。把判断写出来,别套死表。
- 同一块板信号网和电源网线宽**应当不同**——一刀切是反模式。

### 电源网识别(--power-nets)
配电干线(一个网喂多颗 IC / 多个 pad)、已知大电流支路,都该进 `--power-nets`。KRT 原生支持给指定网单独走粗线,这是它的核心能力,别漏用。

### 差分对(--impedance)
名字成对、功能上需要阻抗匹配的网(ADC 差分输入、高速串行等)用 `--impedance` 设目标阻抗。注意:**主 `route` 不做 P/N 等长 / 受控间距**——那是 KRT `route_diff.py` 专用布线器的活,当前 draw-pcb 未包装它。需要严格等长时在 KiCad GUI 手动处理,或单独跑 KRT route_diff。

### 布线顺序(--ordering)
- `inside_out`(KRT 默认):多数板适用,从板内向外布。
- `mps`:拥挤板 / 默认顺序布不通时试。
- `original`:复布场景,保持原网表顺序。
布不通时,换 ordering 是低成本的第一手,优先于改线宽 / 加层。

### via 尺寸(--via-size / --via-drill)
默认通常够用。仅在大电流过孔需要加粗、或 fab 钻孔能力有下限时才调。

## 与其它阶段的衔接
- 布线**必须先过 route-ready 验收**(见 SKILL.md)——烂布局上布线 = 烂地基盖楼。
- 布完线**必须重铺 GND 铜**——见 `references/copper_pour.md`,Phase D 的铜布线后已 stale。
- 布完**必跑 `run_drc`**——KRT 自报布通不等于 DRC 干净;`copper_edge_clearance` 等违例的处理见 `routing.md`。

## 强制前置：读 USER.md 和工作区 CLAUDE.md

讨论开始前必须读这两个文件：

| 来源 | 提取什么 | 怎么用 |
|---|---|---|
| `USER.md §0` | 所属地、时区、语言 | **传给 component-selecting，不在本 skill 重复硬编码** |
| `USER.md §1` | 在手 MCU、电源、负载、示波器、HV 能力 | 决定推荐能不能在用户桌面**测得动** |
| `USER.md §3` | HV 工作能力、焊接能力、空间限制 | 决定 BOM 上限（焊不动的封装不推） |
| `USER.md §4` | 长期研究方向 | 让设计跟主线对齐 |
| `工作区 CLAUDE.md` 工程默认值 | MCU / ADC 位数 / 电源域 / 差分链铁律 | 拓扑必须满足，违反就报警 |

## 落盘收口（exit 三件事，按顺序，做完才交棒）

落盘不是写完 §3/§5 就走。收口要做完这三件，再告知用户进 component-selecting：

### 1. spec 三态完整性（§5 不留裸空格）

每颗 role 的 `目标 spec` cell 必须是 {硬值 │ 范围/约束 │ TBD:原因} 三选一：

- 能现在拍死的 → **硬值**（`Viso≥5kVrms`）
- 该留给市场落点的 → **范围/约束**（`Viso≥3kV(选品落点)`），**不要硬填精确点值**（会鸡生蛋：精确值取决于市面有什么）
- 真填不了的 → **`TBD: <为什么 + 谁来解>`**，绝不留白

> "能填的就填" ≠ "随便空着"——空着也要写清为什么空。`TBD` 的 role 在解掉前不进 Phase 2；其余 role 照常往下，不被它挡。

### 2. 初期可行性评估（self-judgment，**不仿真**）

落盘前自己判断一遍「这回路逻辑上跑不跑得通」，给一个**明确标注【初期评估】**的结论——**不是验证结论，别说得像拍死了**（真正的稳定性/瞬态留 3.5 SPICE，EMC/热留 4.5 check-pcb）。

- 此刻**没有 .kicad_sch、没有 netlist、没有真值** → 不能仿真，只能纸面判断
- 可以从这些角度**感受**（凭判断，不强制逐条打勾）：预算闭不闭合（功率/电压链/电流/热粗算）、有没有破工作区电气铁律（`electrical_invariants.md`）、信号/电源有没有偷渡隔离边界、有没有悬空 role、USER.md 测/焊/买得动不
- 给三态口径之一：✅ 看着能走 / ⚠️ 能走但有残余风险（列出来，写进项目 §9）/ 🚫 纸面就不闭合（回 Phase 1/2 改拓扑或松 spec）
- 残余风险写进项目 §9 未决问题，交给 3.5/4.5 兜底

### 3. 交棒 component-selecting

spec 冻结表就绪 → 告知用户进 Phase 2。envelope 内候选随便挑，下游不 cascade。

## 回环与回退（component-selecting 退回本 skill 时）

选品撞墙退回来是**合法常态**，不是出错。退不退、退多少，**只看回路结构动没动**：

| 选品发生什么 | 归谁 | §5 / STATUS 动作 |
|---|---|---|
| 🟢 纯换等效件（envelope 内，结构没变，spec 没松）| 选品内部 | circuit-design **不动**，状态保持 |
| 🟡 松**安规** spec（隔离/耐压/CTI/温度额定，哪怕结构没变）| 回本 skill **bless** | bless 后改 §5 该 cell；STATUS 记一笔 |
| 🔴 要改**结构**（拓扑/级数/隔离方案/电源域）| 回本 skill **重开** | 受影响 role `spec_frozen → PENDING/TBD`；STATUS 回退记录 |

- 纯**性能** spec（Rds(on) 等）松动归 🟢，选品自己定，不回本 skill
- 🟡/🔴 的触发由 component-selecting 抛出；本 skill 负责**接住**后按上表处理状态
- 🔴 重开时，把这次结构改动**波及到的全部 role** 一起回退（不是只退失败那颗）——结构一变，邻居的 `spec_frozen` 可能成假冻结

# 嘉立创 EDA Pro 自主设计扩展架构

## 定位

`JLCEDA Design Companion` 是本 fork 的执行层。它直接操作当前嘉立创 EDA Pro 工程，保存后的工程、ERC、DRC、3D 预览和制造预览构成设计记录。KiCad 可用于外部仿真、物理分析和选网路由，不能覆盖 EasyEDA Pro 源工程。

扩展采用“计划先行、执行可回滚、结果可审查”的结构。Agent 负责把需求、拓扑、元件选型和约束编译为结构化计划；扩展只接受经过校验的计划并调用 Pro API。这样可保留每次设计变更的可读输入，也能阻止未经解析的自然语言直接改动图纸。

## 已交付模块

### 原理图 Agent

扩展版本 `0.2.0` 已在原理图编辑器提供“打开原理图 Agent...”。它处理版本为 `1` 的设计计划，按顺序完成：

1. 验证字段、坐标、位号、网络名称和图元数量。
2. 用 LCSC C 编号在 EasyEDA Pro 系统库解析全部器件；任何器件无法解析时不会写入图纸。
3. 放置元件，写入位号、BOM/PCB 标志和料号属性。
4. 创建显式导线、网络标识与网络端口。
5. 执行 ERC，读取网表并保存当前原理图。
6. 出现运行时失败时，按创建的逆序删除本次图元并报告回滚结果。

原理图 API 仍标记为 beta，首批真实工程应从小型电阻、电容、连接器测试板开始，并在 EasyEDA Pro 内确认库件、引脚坐标、网络标识与 ERC 结果。

### PCB 协同

已交付的 PCB 模块保留原有能力：板卡数据读取、布线前检查、选网自动布线、走线/过孔回写和 DRC 参数预校验。高电流回路、模拟敏感区、射频、隔离边界和高速链路必须携带明确物理约束，供规则模块和人工审查使用。

## 原理图计划协议

```json
{
  "version": 1,
  "components": [
    {
      "id": "R1",
      "lcsc_id": "C25804",
      "x": 1000,
      "y": 1000,
      "rotation": 0,
      "name": "10k",
      "add_into_bom": true,
      "add_into_pcb": true
    }
  ],
  "wires": [
    { "net": "VIN", "points": [[900, 1000], [1000, 1000], [1100, 1000]] }
  ],
  "net_flags": [
    { "kind": "Power", "net": "VIN", "x": 900, "y": 1000 }
  ],
  "net_ports": [
    { "direction": "OUT", "net": "VIN", "x": 1100, "y": 1000 }
  ],
  "run_erc": true,
  "save": true
}
```

`components` 必须包含唯一的 `id`、有效 LCSC C 编号和显式坐标。`designator` 缺省时使用 `id`。导线必须携带完整的连续坐标点；原型阶段不根据器件名称猜测引脚或坐标。`net_flags.kind` 支持 `Power`、`Ground`、`AnalogGround`、`ProtectGround`，端口方向支持 `IN`、`OUT`、`BI`。

## Agent 编排

```text
需求与约束
  -> 拓扑规划器
  -> LCSC 选型与数据手册核对
  -> 原理图设计计划
  -> Schematic Agent 验证和执行
  -> EasyEDA Pro ERC / 网表
  -> PCB 布局计划
  -> PCB Agent 预检查、布线、DRC
  -> 3D / 制造预览 / BOM-CPL-Gerber 校验
```

规划器输出前应包含：电源树、接口针脚定义、每个元件的 LCSC 编号、位号、封装和额定值、每条网络的端点、板框、层叠、网络类别、间距、线宽、电流、差分阻抗和禁布区。元件可采购性、封装、针脚和数据手册证据应保存在项目目录中，再交给扩展执行。

## 下一批模块

1. **计划编译器**：接收结构化需求与项目约束，生成并比较原理图计划、BOM 和引脚网络表。
2. **器件核验器**：批量查询 LCSC、提取封装和引脚属性，拦截无库件、封装不一致、额定值不足和停产风险。
3. **引脚感知连线器**：由 Pro API 读取放置后的引脚坐标，将逻辑网络端点转换为显式导线并检测短接。
4. **PCB 布局 Agent**：读取网表、板框和约束，先放连接器、功率器件、关键 IC 和去耦，再生成可审查的摆件计划。
5. **规则与物理审查器**：检查电流线宽、爬电距离、回流路径、差分对、长度、热密度、禁布区和制造能力。
6. **ECO 与版本审计**：保存每个计划、解析结果、ERC/DRC、BOM/CPL/Gerber 校验结论，生成可追溯变更单。

## 设计边界

- 扩展不自动提交打样或贴片订单。
- ERC/DRC、3D 和制造预览保留为 EasyEDA Pro 内的确认步骤。
- 物理规则优先于自动布线成功率；规划器缺少约束时必须标记为待补充。
- KiCad 分析结果作为补充证据保存，不回写覆盖嘉立创 EDA Pro 工程。
- 每次计划只作用于当前打开的原理图或 PCB；执行前需在项目目录记录设计目标与约束。
